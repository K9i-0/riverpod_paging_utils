# OSS Design Review Skill

OSSライブラリのサンプルアプリとして、プロフェッショナルな印象を与えるUIデザインに改善します。

## 目的

OSSの利用者がサンプルアプリを見た時に：
- ライブラリの品質の高さを感じる
- 実装の参考にしたいと思える
- プロダクションでも使えそうだと感じる

## アーキテクチャ

```

```

## フィードバックループ

1. **Subagent**: シミュレーターでスクリーンショットを撮影
2. **Subagent**: スクリーンショットのパスをメインセッションに報告
3. **メインセッション**: 画像を確認し、改善点をフィードバック
4. **Subagent**: フィードバックを元にコードを修正
5. **Subagent**: ホットリロードで反映
6. **繰り返し**: 1-5を満足するまで繰り返す

## レビュー観点（メインセッション用）

OSSサンプルアプリとしてプロフェッショナルな印象を与えるために：

1. **視覚的階層** - 情報の優先順位が一目で分かるか
2. **タイポグラフィ** - フォントサイズ・ウェイトのバランス、可読性
3. **カラーパレット** - 統一感、アクセシビリティ、モダンな配色
4. **余白とスペーシング** - 窮屈さがなく、適切な余白
5. **コンポーネントの一貫性** - ボタン、カード、アイコンの統一感
6. **全体的な印象** - 「このライブラリは信頼できる」と感じるか

## 終了条件

以下を満たしたらループ終了：
- 主要な改善点がすべて対応された
- UIがプロフェッショナルな印象を与える状態になった
- メインセッションがOKを出した
